# Przewodnik dla nowych i przyszłych studentów edytorstwa

Repo startowe do ćwiczeń z:
- pracy zespołowej,
- GitHub Pages,
- commitów,
- pull requestów,
- korzystania z AI przy tworzeniu treści i prostego kodu HTML/CSS.

## Proponowany podział na grupy

1. **Grupa 1** – nagłówek, hero, nawigacja
2. **Grupa 2** – sekcja „O kierunku”
3. **Grupa 3** – sekcja „Rekrutacja / przygotowanie do studiów”
4. **Grupa 4** – sekcja „Narzędzia”
5. **Grupa 5** – FAQ i stopka

## Zasady pracy

- Nie pracujemy bezpośrednio na `main`.
- Każda grupa tworzy własny branch, np. `feature/grupa-1-hero`.
- Każda grupa robi przynajmniej 3 sensowne commity.
- Zmiany trafiają do `main` przez pull request.
- Przed merge'em sąsiednia grupa robi krótkie review.

## Jak włączyć GitHub Pages

1. Wejdź w **Settings** repozytorium.
2. Otwórz sekcję **Pages**.
3. Wybierz źródło: **Deploy from a branch**.
4. Ustaw branch: `main`, folder: `/root`.
5. Zapisz ustawienia.

## Proste prompty do AI

### Do treści
> Napisz krótki, prosty tekst na stronę internetową dla kandydatów na studia z edytorstwa. Styl ma być przyjazny, konkretny i zrozumiały.

### Do HTML
> Przygotuj prostą sekcję HTML z nagłówkiem, akapitem i listą punktowaną. Kod ma być czytelny i łatwy do wklejenia do istniejącej strony.

### Do CSS
> Zaproponuj prosty, estetyczny styl CSS dla strony informacyjnej o studiach. Użyj jasnych kolorów i nowoczesnego układu.

### Do poprawy
> Wyjaśnij, co robi ten fragment kodu, a potem uprość go tak, żeby był zrozumiały dla osoby początkującej.
